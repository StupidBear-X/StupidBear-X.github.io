0X02:基础

linux 内核和发行版的区别：平常所说的linux指的是linux 的内核，发行版就是厂商将内核和软件包打包的集合。常见的发行版又suse，open suse ，redhat，ubuntu，centos等。内核被不同的厂商开发定制打包就出现了不同的发行版。

常见的操作系统有windows ，Android，linux，mac，unix等。mac和Android都是源于linux。linux 源于unix。unix 出自贝尔实验室的文件管理系统。

![image-20191212200214588](/Users/xiong/Library/Application Support/typora-user-images/image-20191212200214588.png)

Linux和unix的关系

![image-20191212201927644](/Users/xiong/Library/Application Support/typora-user-images/image-20191212201927644.png)

为什么linux安全：首先linux 是开源的，开源的同时就会有更多的人来完善和修改漏洞。

windows 和 Linux比较：linux 免费，安全，对处理高并发十分给力，多命令行操作，可以自由定制，多适用于服务器。

BIOS ： 基本输入输出系统，是一组固化到计算机内主板上一个ROM芯片上的程序，保存着计算机最重要的基本输入输出的程序，系统设置信息，开机上电自检程序和系统启动自举程序。为计算机提供最底层，最直接的硬件设置和控制。

虚拟机网络连接 ：三种形式的说明：

1、桥连接（桥接）： 主机和虚拟机配置同一网段的IP地址，缺点是如果使用人数过多，IP地址会不够用（IP地址容易冲突）。

2、 NAT模式：主机会虚拟出一个网卡，虚拟网卡的ip和虚拟机的网卡ip再用一个网段，这样好处是不会占用原有主机网卡ip段的IP地址。不会造成ip冲突。

3、 仅主机模式 ： linux 是一个独立的主机，不可以访问外网。

再再再注意 ： linux 分区！！！！！！

1、至少要有三个分区，

根分区（挂载点是/ ，其他的都给根分区，文件类型是ext4），

swap分区（没有挂载点，文件类型是swap，当系统内存不够用时，暂时替代为物理内存，一般为物理内存的两倍），

boot分区（存放系统启动的引导程序，挂载点是/boot，文件类型是ext4）。

kdump ： 内核崩溃转储机制，会占用一定内存，在实际生产环境中要勾选使用。

0X03:目录结构

linux 的目录结构是一个树形结构，只有一个根目录（ / ）。

再次强调一次，在linux 中，一切皆是文件。设备也是文件。

![image-20191216163928408](/Users/xiong/Library/Application Support/typora-user-images/image-20191216163928408.png)

  • / 根目录（有且只有一个根）
  • /dev 存放可以驱动的设备文件，将硬件映射成一个文件。
  • /bin bin是binary的缩写，常用的指令都在这里！（⚠️）
  • /sbin 存放系统管理员使用的系统管理程序
  • /opt 主机额外安装的一些软件存放在这里
  • /usr/local 这是另一个给主机额外安装软件的安装目录，一般是通过便衣源码方式安装的程序
  • /var 这个目录中存放着不断扩充的东西，习惯将经常被修改的目录放在该目录下，包括各种日志文件
  • /selinux 类似linux的360，一种安全子程序，控制程序只能访问特定文件
  • /lib 这个目录里存放着系统最基本的动态链接共享库，其作用类似于windows的DLL文件，几乎所有的应用程序都需要用到这些共享库。
  • /home 用户的主目录，在Linux中，每个用户都有一个自己的目录，一般该目录名是以用户的账号命名的。（⚠️）
  • /etc 所有的系统管理所需要的配置文件和子目录my.conf（⚠️）
  • /usr 用户的很多应用程序和文件都放在这个目录下，类似于windows 下的 program files 目录（装软件的）（⚠️）
  • /user
  • /lost+found 这个目录一般是空的，当系统非法关机后，这里就存放里一些文件。
  • /boot 存放的是启动linux时的一些核心文件，包括一些连接文件以及镜像文件。（一般是系统引导用到的）
  • /proc 一个虚拟的目录，是系统内存的映射，访问这个目录来获取系统信息。（⚠️）
  • /srv service 的缩写，存放一些服务启动后需要提取的数据（⚠️）
  • /sys 该目录下安装了2.6内核中新数显的一个文件系统sysfs（⚠️注意，proc && srv && sys这三个目录一般不要修改，有可能导致系统奔溃）
  • /tmp 存放一些临时文件
  • /media linux系统会自动识别一些设备，如u盘，光驱等，识别后会把识别到的设备挂载到这个目录下（⚠️）
  • /mnt 系统提供该目录是为了让用户 临时挂载别的文件系统的，可以将外部的存储挂载到/mnt/上，然后进入该目录就可以查看里面的内容了（可以识别其他的文件系统）（⚠️）
  • /root 该目录为系统管理员，也称作超级权限者的用户主目录（⚠️）
  • （⚠️）总结一下： linux 的目录中有且只有一个根目录；linux 的各个目录存放的内容是规划好的，不要乱放文件；linux 是以文件的形式管理我们的设备，在linux 中一切皆为文件；linux的各个文件目录下存放什么内容，必须有一个认识。

Vi 和 Vim基本使用：

所有linux 都会内含vi编辑器；vim可以看作是vi的增强版，可以编写程序。

⚠️vi 和 vim 的三种常见模式

1 正常模式

​ 在正常模式下，可以使用快捷键。以vim打开一个档案就直接进入一般模式（这是默认模式），在这个模式中，你可以使用「上下左右」按键来移动光标，你可以使用「删除字符」和「删除整行」来处理档案内容，也可以使用「复制，粘贴」来处理你的文件数据。

2 插入模式/编辑模式

​ 按下i，I，o，O，a，A，r，R等任何一个字母后才会进入编辑模式，常用的按i键。

3 命令行模式

​ 在这个模式中，可以提供你相关指令，完成读取，存盘，替换，离开，Vim，显示航号等的动作则是在此模式中达成。

⚠️

  • tar -zxvf 文件名
  • ll （显示当前文件夹内文件的详细信息）
  • reboot （重启系统）
  • useradd xxxxx 添加用户xxxx
  • userdel xxxxx -r。 删除用户xxxx
  • ifconfig 查询ip地址
